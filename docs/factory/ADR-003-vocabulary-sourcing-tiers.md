# ADR-003 — Content & animation vocabulary: sourcing tiers (A/B)

**Date:** 2026-06-22 · **Status:** Accepted — the standing reuse map for all vocabulary buildout.

## Context

A real editor (DaVinci/Kdenlive/AE) exposes hundreds of transitions, effects, shapes, titles, color tools, etc. We want that breadth **without inventing it**. Almost all of it is available free — from Remotion's own packages and the OSS ecosystem. The deciding constraint for *us* is **determinism** (golden rule #1): DOM/SVG/CSS rendering is byte-deterministic on any GL backend and disk-safe; WebGL/GPU rendering is non-deterministic across runs *and* triggers the software-GL CacheStorage disk-balloon (DECISIONS 2026-06-22).

## Decision

**Our families are sockets; free libraries are plugs.** We do not author hundreds of effects/transitions — we adopt the catalogs and expose them through the channels already designed (`generator`, `effects[]`, `transition`, shape/text layers). Everything is classified into two tiers:

- **Tier A — adopt now (deterministic, disk-safe, DOM/SVG/CSS/CPU).** The default. Build these into the families directly.
- **Tier B — opt-in, gated (free but WebGL/GPU → non-deterministic + disk-balloon risk).** Allowed only behind an explicit flag and only once **deterministic GPU capture** is solved (capped/disabled Chromium cache + accepting/again-pinning GPU output). Until then, Tier B is for non-deterministic previews only.

**Rule:** prefer Tier A. A feature only goes Tier B if it genuinely cannot be done in SVG/CSS/CPU.

## Build backlog (nothing skipped; each = a verified increment)

Each item is built into its family, **adopting the Tier-A source**, and verified per `verify-render` (typecheck + cross-process decoded-stream determinism + visual frames). Tier-B items are deferred behind the GPU-determinism gate.

1. ✅ **Shapes & paths (DONE 2026-06-22)** — adopted `@remotion/shapes` (rect/circle/ellipse/triangle/star/polygon/pie/heart primitives) + `flubber` (path **morph**, differing point counts) + `@remotion/paths` (`getBoundingBox`); first-class `shape` layer in `src/render/ShapeLayer.tsx` with solid/linear/radial **gradient** fill (deterministic `<defs>` ids from layer id), **stroke**, the standard `{a,k}` transform, and a `morph` channel whose `fill` interpolates alongside — all StyleKit-eased (never linear). Wired through `<Scene>` so shapes get parallax + §11.1 shading. Verified per `verify-render`: typecheck clean + byte-identical decoded-stream MD5 across two cold render processes + frames reviewed (`projects/shapes-demo`: primitives + a mid-morph star→heart).
2. ✅ **Effects channel (DONE 2026-06-22 — authored AS A PLUGIN)** — the per-layer `effects[]` stack backed by **SVG/CSS filters** + **motion blur**, shipped as the **`core-effects` plugin** populating the engine's `effects` extension point (a *plugin*, not a core edit; ADR-005 "sockets/plugs"). `EngineAPI.registerEffect(kind, impl)` writes into the `effects` registry; each `EffectImpl` validates its own params (in-plugin Zod) and `build(params, frame)` returns a deterministic `EffectContribution` over three composing channels — `filterPrimitives` (chained SVG `<filter>` primitives), `cssFilter` (concatenated CSS `filter` fragments), and `wrap` (layer-subtree wrappers). Registered kinds: **blur** (feGaussianBlur), **glow** (feGaussianBlur+feColorMatrix+feMerge bloom), **drop_shadow** (CSS drop-shadow), **color_grade** (CSS brightness/contrast/saturate/hue-rotate), **turbulence**/**displace** (feTurbulence+feDisplacementMap, animatable phase), **vignette** (radial-gradient overlay wrap), **grain** (feTurbulence+feColorMatrix+feBlend), **motion_blur** (`@remotion/motion-blur` `<Trail>` wrap). The compositor (`src/render/effects.tsx`) walks a layer's `effects[]` IN ORDER, chains SVG primitives inside ONE `<filter id="fx-<layer.id>">` (deterministic id from layer id, prior stage's `result` feeds the next), concatenates CSS fragments, and applies `wrap`s outermost — composing cleanly ON TOP of the default-on §11.1 shading drop-shadow + parallax (separate wrapper, never fights the single CSS `filter` slot). All Tier-A (no WebGL), frame-deterministic on the CPU raster default. Verified per `verify-render`: `tsc --noEmit` clean + `projects/effects-demo` (exercises every kind) byte-identical across two cold processes + frames eyeballed (glow bloom + motion_blur trail visible).
3. ✅ **Transitions expansion (DONE 2026-06-22 — authored AS A PLUGIN)** — the scene-boundary transition mapping moved OUT of core `Composition.tsx` (the old hardcoded switch with a fade fallback) INTO the engine's `transitions` extension point (previously an empty stub), populated by the **`core-transitions` plugin** via `api.registerTransition`. The core compositor now resolves `transitions.get(scene.transition_in.kind).build(ctx)` for a `@remotion/transitions` `TransitionPresentation` (falling back to fade only on a bare engine). Tier-A `@remotion/transitions` DOM presets are adopted, NEVER reimplemented (`fade`/`wipe{direction}`/`slide{direction}`/`iris{width,height}`); the kinds Remotion has no preset for are small custom `TransitionPresentation`s using native primitives: **mask** (soft-edged SVG radial-mask reveal of the entering scene), **morph-match**/**match-cut** (a `flubber`-morphed shared-element shape bridging the cut over a crossfade, `from`/`to`/`tint` path params), **camera-continuous** (one uninterrupted translate+scale shared across the boundary). All timed by `linearTiming` (pure fn of frame). GL Transitions (123 GLSL) remain Tier-B (gated, item 11). Verified per `verify-render`: `tsc` clean + the `m2-demo` mega-demo (iris + mask + wipe) byte-identical across two cold processes.
4. ✅ **Generators expansion (DONE 2026-06-22)** — added four procedural generators on top of scatter + bead-string: **`water`** (stacked translucent scrolling wave bands + foam), **`particles`** (looping drift/rise/fall field with sway + fade), **`fire`** (layered flickering teardrop flames + warm gradient + rising smoke), **`crowd`** (jittered-grid field of tiny head+body creatures with idle bob + blink-squash). Each is a Tier-A `FC<GeneratorComponentProps>` in its own `src/generators/<name>.tsx` (own in-file Zod schema, `.strip()`, all fields defaulted; `resolveFill` for palette tokens), seeded RNG only (`mulberry32`/`mixSeed` + `simplex-noise`), a pure `renderX(props)` split from the hook wrapper, and a full-size absolute `<svg>`. Every animated value is a **closed-form function of (frame, index, seed) with a looping lifecycle** — no cross-frame accumulation (Remotion renders frames out-of-order across workers). Each honors a shape budget (clamp + `console.warn`, no silent truncation). Registered in `src/generators/registry.ts`. Verified: `tsc --noEmit` clean + each renders, is deterministic at a fixed frame, and animates across frames.
5. ✅ **Text/typography (DONE 2026-06-22)** — first-class generic `text` layer (`src/render/TextLayer.tsx`) with a vendored deterministic font (`@font-face` + `delayRender`, no CDN), `@remotion/layout-utils` box-fit, and kinetic presets (fade/rise/stagger/typewriter/count_up) via `interpolate`/`spring`. (Narration-synced text + `@remotion/captions` is M3.)
6. ✅ **Color-script (DONE 2026-06-22)** — `src/pipeline/color-script.ts`: a beat may carry a `mood` (resolved to a named `library/palettes/*` token map) and/or an inline `palette`; lowering layers them over the resolved stylekit base (most-specific-last) into a per-scene palette, and INTERPOLATES adjacent scene palettes in **OKLab** across each transition via **`culori`** (`culoriInterpolate` — never hand-rolled). The per-scene palette (diff vs base) rides the Scene IR (`scene.palette`) and the renderer reads it through the existing `defs.stylekit` palette path, so tokens flow into shading + fills. Pure/deterministic (fn of palettes + t).
7. ✅ **Asset vocabulary (DONE 2026-06-22)** — the **`core-objects` plugin** contributes a generic `object` PROVIDER (a peer of `blob-creature`): a small `ObjectSpec` → flat-vector SVG markup for simple props (deterministic, no network, vendored shapes). A prop is cast like any rig (`ref` → a library object entry) and placed standalone or `attach`-ed to a rig mount. (Iconify/unDraw/Open Peeps bulk ingest as library entries remains an incremental follow-on.)
8. ✅ **Compositing (DONE 2026-06-22; track-matte clipPath FINISHED 2026-06-23)** — `src/render/compositing.tsx` applies, generically in the layer wrapper on top of §11.1 shading + `effects[]` + parallax: a per-layer **blend mode** (CSS `mix-blend-mode` + `isolation`) and a **track matte / mask**. The reliable matte sources: a non-inverted **ASSET** image (`matte.ref` → CSS `mask-image`) and — NEW 2026-06-23 — a **SHAPE-source LAYER matte** (`matte.from` → a `shape` layer): the source shape's screen-space path geometry (`shapeClipGeometry` in `src/render/ShapeLayer.tsx`) is emitted as a composition-sized SVG stencil and applied as a STATIC `mask-image` **data-URI** (NOT a live SVG `clip-path` — a live clip-path re-rasterized its anti-aliased edge per frame with sub-pixel drift, ~51 dB, byte-identical under `renderStill` but NOT across cold `renderMedia` processes; the data-URI mask is the proven byte-identical asset-matte path). A general layer matte (arbitrary non-shape sibling) and inverted mattes still degrade GRACEFULLY (show unmasked + warn once) because headless Chromium renders `<foreignObject>` HTML into an SVG `<mask>` as empty. This SUPERSEDES the earlier deferred no-op. Nested `clip`s (§13.3) + the `environment` backdrop convention + compositional rig **`attach`/`parts` (§8.1)** are all live. Runtime primitives only, never reimplemented; static styles → byte-deterministic.
9. ✅ **Audio (M3) — DONE: narration 2026-06-22; captions + SFX + music/ducking 2026-06-23.** ✅ **Narration** — OFFLINE TTS (engine-abstracted: **espeak-ng** default/deterministic, **Coqui XTTS v2** via an isolated venv with espeak-ng fallback) → a **content-addressed, cached wav** (`src/cli/narrate.ts`, skip-if-exists) at BUILD (golden rule 2); `src/cli/narrate-pass.ts` emits `kind:"narration"` `audio[]` cues; `src/render/Composition.tsx` `NarrationTrack` plays each `<Audio>` in a `<Sequence from={cue.at}>`; `renderMedia` muxes (h264+aac). ✅ **Captions (A1)** — `captions[]` cues DERIVED from the same `say` text + cue window (deterministic, no whisper) → `src/render/CaptionTrack.tsx` (`line`/`words` cadence). ✅ **SFX (A2)** — `show[].sfx`/`beat.sfx[]` → `kind:"sfx"` cues at the event frame (`src/cli/sfx-pass.ts`); palette synthesized by ffmpeg lavfi into the shared `library/sfx/` cache (`src/cli/sfx.ts`). ✅ **Music + ducking (A3)** — a `music` bed (`library/music/` ffmpeg beds or asset ref) → one `kind:"music"` cue (`src/cli/music-pass.ts`), `<Loop>`-tiled, ducked per-frame from `gain`→`duck` while a narration window overlaps (`duckingAt` in `Composition.tsx`; the reserved `<Audio volume>` hook is now live). No audio primitive reimplemented (mixing/looping all Remotion). Verified per `verify-render`: BOTH the decoded video AND audio streams byte-identical across two cold processes AND the muxed audio AUDIBLE (`volumedetect` mean −23 dB ≫ −60 dB). **Remaining:** whisper word-level alignment → precision captions + lip-sync visemes.
10. ✅ **Data-viz (DONE 2026-06-22)** — the **`core-dataviz` plugin** contributes a `chart` GENERATOR (bar/line/pie) built on **`d3-shape`/`d3-scale`** → SVG (Tier-A, deterministic, no canvas/WebGL), with an animatable **draw-on** (delay/duration/stagger/easing, closed-form fn of frame), palette-token fills, optional axes, and its own in-plugin Zod params. Maps (D3-geo) deferred.
11. **Tier-B enablement (gated)** — deterministic GPU capture (capped Chromium cache) → unlock GL Transitions / PixiJS filters / postprocessing / three / skia.

## Consequences

- The vocabulary grows by **adopting catalogs into families**, not authoring effects — fast, free, and consistent with ADR-001 (engine generic; families pluggable).
- Tier-A items are shippable + deterministic immediately; Tier-B is a single future enablement (GPU-determinism) that unlocks a large catalog at once.
- Quick wins flagged: stop hand-rolling shapes (`@remotion/shapes`) and morph (`@remotion/paths`).

## Runtime vs compiler boundary (the "are we reinventing Remotion?" rule)

**Remotion is the RUNTIME; our IR is the compiler TARGET.** Remotion natively provides the deterministic frame model (`useCurrentFrame`), the animation primitives (`interpolate`, `spring`, `Easing.bezier`, `interpolateColors`), sequencing (`Sequence`/`Series`/`@remotion/transitions` `TransitionSeries`), media/layout (`AbsoluteFill`/`Img`/`Audio`/`staticFile`), and the render backend (`bundle`/`renderMedia`/`renderStill`). Its "declarative" form, however, is **React + props (code)** — it has **no serializable, generatable, validatable film *document***.

That gap is the ONLY reason our wrapper exists: a **serializable Scene-IR format** (Lottie-superset `{a,k}`, Zod-validated, content-hashable), a **script → Story IR → Scene IR → frames compiler** (pure staged passes), a **content-addressed library/lockfile**, determinism tooling (cross-process verify, golden-diff), and the **AI-emits-*data* boundary** (AI writes validated IR, never code or frames — the founding motivation). The Scene IR is passed **verbatim as `inputProps`** to one generic compositor — there is no runtime translation layer.

**Rules:**
1. **Never reimplement a Remotion primitive.** Interpolation, easing, springs, sequencing, transitions, color interpolation — call Remotion (e.g. `eval.ts` reads IR keyframes and calls Remotion `interpolate`; `EASINGS` are bezier tuples fed to `Easing.bezier`; overshoot uses `spring()`). The render layer is a THIN IR→Remotion adapter.
2. **Borrow capability from Remotion's ecosystem first** (Tier A): transitions→`@remotion/transitions`, text→`@remotion/google-fonts`+`layout-utils`, shapes→`@remotion/shapes`, footage→`@remotion/lottie`/`<Video>`, motion-blur→`@remotion/motion-blur`. Add a non-Remotion lib only where Remotion has none (e.g. `culori` for OKLab, `flubber` for morph, `d3-shape` for charts).
3. **Let Remotion own playback timing.** The compiler computes only the bookkeeping it needs (total `duration_frames`, per-scene `at`); `TransitionSeries`/`Sequence` own the actual overlap/playback — do not re-derive it.
4. **Trap to avoid:** if the goal were "write a React composition per video," almost all the wrapper would be redundant with Remotion. The wrapper earns its keep ONLY as the *data-driven factory* (videos as generated, reusable, reproducible DATA). Keep the runtime layer minimal; put the weight in the compiler/IR/library/plugins, never in re-doing Remotion.
