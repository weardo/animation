# Factory Decisions Log

Append-only. One entry per significant decision/learning: **Context → Decision → Rationale → (Supersedes?)**. Maintained via the `refine-standard` skill.

---

## 2026-06-22 — Render host: Remotion
**Context:** Need code-driven, deterministic, audio-capable, batch-able rendering with minimal plumbing; solo dev (license a non-issue).
**Decision:** Remotion is the host/orchestrator/encoder; Scene IR is its `inputProps` (no translation layer). Other renderers run *inside* it.
**Rationale:** Built-in deterministic render + audio + FFmpeg mux + batch; most LLM-authorable (React); deletes the need for a custom frame-capture loop.

## 2026-06-22 — Rigs: DragonBones with full mesh deformation
**Context:** Need identity-stable, code-generable skeletal rigs with organic deformation; no editor lock-in / per-seat cost.
**Decision:** Adopt DragonBones JSON format + `pixi-dragonbones` runtime in a Pixi canvas inside Remotion (committed render path), with **full mesh deformation (FFD)** in scope.
**Rationale:** Free/MIT, standard, IK + mesh deform built in; FFD covers bendy/blobby warping. Pixi-canvas chosen over pure-SVG specifically to get mesh deform. Supersedes the earlier "Spine-derived custom schema" idea.

## 2026-06-22 — Scene IR is a Lottie superset; Zod is the single source of truth
**Context:** Avoid inventing an animation IR.
**Decision:** Scene IR adopts Lottie's `{a,k}` property/keyframe model (round-trippable) and extends only where Lottie can't reach (rig refs, camera-parallax, generators, audio, morph/effects). Define all IR in **Zod** → TS types + runtime validation + JSON-Schema (for the future LLM front-end).
**Rationale:** Reuse a Linux-Foundation standard; one definition does three jobs.

## 2026-06-22 — Organic motion via a generator family
**Context:** Keyframes + morph can't express water, fire, neuron bead-strings, crowds.
**Decision:** A `generator` layer family — parametric components, pure functions of `(params+seed+frame)`, in an extensible registry. Water = one generator; neurons = another.
**Rationale:** Don't special-case effects; one mechanism, deterministic, future-proofs fire/smoke/clouds/crowds.

## 2026-06-22 — Library: content-addressed, versioned, lockfile-pinned; fractal reuse
**Context:** Reuse characters/clips/scenes across many videos as the library grows, without changing old videos.
**Decision:** `name@version` → content hash (`object-hash`), `library/index.json` catalog, `animation.lock` pinning. Reusable kinds at every granularity (parts → rigs → presets → clips → scene-templates → videos) via Remotion nested compositions.
**Rationale:** Package-manager patterns (not a custom registry); lockfile reconciles "growing library" with "deterministic renders."

## 2026-06-22 — Effects/post and audio are parallel, event-driven stacks
**Context:** Need After-Effects-style polish and sound design without losing sync.
**Decision:** Layer `effects[]` + composition `post[]` (reuse SVG/Pixi/Remotion FX, `@remotion/transitions`, `@remotion/motion-blur`); audio mixing + SFX-from-animation-events. Both driven by the same animation events. Motion blur is a StyleKit default.
**Rationale:** One event timeline drives picture and sound → coherence; everything reserved-in-IR-now, built M2/M3.

## 2026-06-22 — A video is a reproducible PROJECT bundle (standards-grounded, not invented)
**Context:** Video artifacts were scattered (source in examples/, scene.json ephemeral in .cache/, a single global animation.lock, out.mp4 at root) — no reproducible per-video unit. Checked for standards rather than inventing.
**Decision:** Adopt the convention shared by **OpenTimelineIO `.otiod`/`.otioz` bundles** and **dotLottie `.lottie`**: a root **manifest** + a **timeline/scene doc** + organized **media**, in a working DIRECTORY form (zip-able later). A project = `projects/<id>/{project.json, story.yaml, scene.json, project.lock, media/}`. CLI is project-centric: compile (`render <story> --project <id>`) vs reproduce (`render <id>` → renders the pinned scene.json, verified byte-identical). Per-project lock replaces the global one. `scene.json` promoted from throwaway cache → tracked, deterministic engine input.
**Rationale:** library : project :: npm package : app (third pillar after engine + library, ADR-001). The dir-bundle is OTIO-export-able and shareable as a zip (future `factory bundle`). Tracked: manifest/source/scene/lock (reproducibility); media/ gitignored (regenerable). Sources: OTIO `.otiod`/`.otioz`, dotLottie spec.

## 2026-06-22 — Software GL (swiftshader) balloons the disk; use 'angle' for procedural scenes
**Context:** Renders filled the root partition (~26GB) then crashed, then freed — for both the user and the agent. Root cause: `chromiumOptions.gl: 'swiftshader'` (software renderer) ballooned Chromium's Service-Worker **CacheStorage** to ~26GB during render, held open in the chrome temp profile (released on process death → the "fills then frees" symptom). Caught by monitoring `df` live during a render and killing before crash.
**Decision:** Use `gl: 'angle'` (hardware) for the render. Procedural characters are pure SVG/DOM, so determinism is DOM-based (not GL-based) — 'angle' is byte-identical across cold runs AND fast (~28s vs crash). Verified: two renders decoded-MD5 identical, disk flat at 31G. Software GL was only needed for WebGL/DragonBones FFD determinism — another reason to prefer the procedural provider. A future WebGL-determinism need must first solve the cache bloat (capped/disabled Chromium cache).
**Process lesson (→ verify-render):** when a render hangs/crashes, monitor `df -h /` live; "fills then frees on crash" = a process holding large unlinked temp open. Orphaned renders survive killing the npm wrapper — `pkill -9 -f compositor-linux`/`chrome-headless` too.

## 2026-06-22 — blip rebuilt procedurally; first real "procedural" provider (ADR-001)
**Context:** The code-generated DragonBones `blip` rendered broken — the FFD body mesh collapsed to a wedge and limbs misaligned. The *procedural art* (atlas parts) was actually clean; the breakage was 100% in the DragonBones skeleton/FFD assembly (hand-synthesizing a vendor mesh without the editor is brittle).
**Decision:** Implement the ADR-001 `procedural` rig kind: `RigDef.kind` enum (`dragonbones|procedural`), library `format` → provider in `toRigDef`, and Scene.tsx dispatches to `<ProceduralRig>` (pure-SVG primitive-shape character, idle/breathe/blink/wave) vs `<RigLayer>` (DragonBones). `blip` is now procedural (`proc://blip`); removed the broken DragonBones blip files + `gen-blip-rig.mjs` + FFD probes. Procedural rigs are deterministic-by-construction AND fast (no Pixi/WebGL/swangle). Visually verified clean; frame byte-identical across processes.
**Process lessons (→ verify-render skill):** (1) ALWAYS look at a frame — determinism ≠ correctness (I called the broken blip "verified" off determinism alone). (2) Remotion caches the webpack bundle — `rm -rf node_modules/.cache .remotion` if a code change doesn't show. (3) The pipeline Scene-IR cache key omits library state — `rm -rf .cache` after library/provider changes.

## 2026-06-22 — M2.1 Shading & Depth compositor implemented (foreground, not workflow)
**Context:** The M2 workflow stalled with slow render/verify loops (6–10 min idles); finished M2.1 in the foreground instead.
**Decision:** Shading lives in `src/render/shading.tsx` (`SceneLook`, `ContactShadow`, `objectFilter`, resolvers) + `Scene.tsx`. Scene `light` defaults on (StyleKit `DEFAULT_LIGHT`); per-layer `shading` defaults on (`DEFAULT_SHADING`), background-exempt. Components: directional light wash + vignette (screen-space), per-object contact shadow (camera-tracked), silhouette-following rim/AO/glow via CSS `drop-shadow` (works on SVG layers AND the Pixi canvas). Visually verified on `blip`; a shaded frame is byte-identical across two separate processes (determinism held — static styles + deterministic anchor).
**Deferred (honest):** true per-silhouette form-shade overlay (approximated by the scene wash + rim for now); gradient fills (typed, not yet rendered); far-layer atmospheric tint. **Process lesson:** a tight build→render→look loop is far faster in the foreground than in a render-heavy background workflow (logged under ADR/standard).

## 2026-06-22 — Shading & Depth is compositional, not post-processing
**Context:** Kurzgesagt depth comes from each object carrying supporting gradient shapes (contact shadow, form shade, rim, AO, glow) consistent with one scene light — not from per-object filters.
**Decision:** Add a Shading & Depth model (spec §11.1): a scene-level `light` as single source + a default-on per-layer `shading` that auto-generates supporting gradient shapes from silhouette + light + `z`; gradients are first-class animatable fills. Reuse SVG gradients/lighting filters + Pixi filters. M2 (look); fields reserved in Scene IR now.
**Rationale:** A single scene light makes the whole scene read as one coherent lit space; auto-derivation avoids hand-authoring shadow shapes (the manual labor we're eliminating). Quality floor → default-on. **Supersedes** the earlier assumption that StyleKit's drop-shadow/glow `effects` filters covered Kurzgesagt depth — they don't.

## 2026-06-22 — Object detail is a render-budget problem; split by behavior
**Context:** Kurzgesagt objects are hundreds of tiny shapes each. Treating all detail as live SVG DOM would crater laptop render times.
**Decision:** Add a `scatter` generator for procedural detail (seeded, poisson-disc) and a detail×performance strategy (spec §10.1): bake static high-count detail (cached/content-hashed), Pixi `ParticleContainer` for animated high-count, `<symbol>`/`<use>` for motifs, bake surface-bound detail into rig textures (deforms via FFD), and a per-scene shape budget with logging (no silent truncation). Added as risk #6.
**Rationale:** Detail density is a rendering-budget question, not an art one; splitting by behavior keeps it deterministic and laptop-viable. Authored multi-shape assets already work; the strategy is M2 + ongoing.

## 2026-06-22 — Deterministic WebGL render recipe (corrects the M1 spike)
**Context:** M1 built and self-reported "byte-identical determinism PASS." Independent cross-run verification (a fresh render in a new process, comparing the *decoded video stream*) found the opposite: ~all frames differed with hardware GL, then ~3/150 differed after switching GL — the in-session check only compared two back-to-back renders, which masked it.
**Decision:** The verified recipe for byte-identical Pixi/DragonBones output across cold runs is: (1) `chromiumOptions.gl: 'swangle'` (software SwiftShader — hardware `'angle'` is non-deterministic across runs), (2) `preserveDrawingBuffer: true` on Pixi init, (3) **synchronous** `continueRender` after `app.render()` (rAF gating fights Remotion's frame clock). Result: two cold renders byte-identical, 0/150 frames differ.
**Rationale:** GPU float variance + a paint-commit race were the two non-determinism sources. **Supersedes** the spike's "angle + rAF retires the determinism risk" claim.
**Process lesson (→ verify-render skill):** test determinism across SEPARATE process invocations and compare the DECODED VIDEO STREAM (`ffmpeg -map 0:v -f md5`), not the container (it carries a wall-clock `creation_time`). Updated the skill accordingly.

## 2026-06-22 — Visual-language systems: transitions/match-cuts, kinetic typography, color-script
**Context:** Kurzgesagt storytelling relies on seamless idea-to-idea transitions (match-cuts), bold animated text, and an emotional color arc across beats.
**Decision:** Add three first-class systems (spec §11.2–11.4): (1) transitions incl. `morph-match`, `match-cut` (shared element continuity via `match:{from,to}`), and `camera-continuous` — reuse `@remotion/transitions` + `flubber`; (2) a `text` layer (split from `asset`) with kinetic-reveal presets, auto-fit (`@remotion/layout-utils`), count-up, and later narration sync (Whisper + `@remotion/captions`); (3) a color-script where Story-IR beats carry `mood`/`palette`, Scene-IR palette tokens are the single source for all fills/gradients/`light.color`, and palettes interpolate across transitions in OKLab (`culori`/`d3-interpolate`).
**Rationale:** All three generalize existing mechanisms (transitions/morph, layers+stagger, palette tokens) rather than adding subsystems; reuse-first; deterministic. M2 (M3 for narration-synced text). Fields reserved in IR now.

## 2026-06-22 — M2.2 Multi-scene storytelling + transitions implemented
**Context:** The pipeline lowered a Story IR into ONE scene; storytelling needs a multi-BEAT story to become N SEQUENCED scenes on the global timeline, rendered as one continuous film with transitions between them (spec §11.2).
**Decision (how it composes):**
- **Beats → scenes (1:1).** `src/pipeline/lower.ts` lowers each Story-IR beat into one Scene-IR scene. Beat duration resolves from `duration` (`{seconds}`/`{frames}`/bare-seconds → frames via fps), falling back to `DEFAULT_BEAT_SECONDS` (4s). The single-beat M1 case with no duration/override keeps its 5s `M1_CONFIG` slice, so M1 byte-output is unchanged. A beat's `show[]` gates whether its bead-string generator layer is emitted, so the film reads as a story (intro shows just the character; the network appears when introduced).
- **Sequencing on the global timeline.** Cumulative `at`: `at[i] = at[i-1] + dur[i-1] − overlap[i]`, where `overlap[i]` is the frames scene[i]'s `transition_in` overlaps the previous tail (0 for `cut`/none). Each beat's `transition` lowers to the scene's `transition_in` (the first beat's is dropped — nothing precedes it).
- **Transition composition.** `src/render/Composition.tsx` renders the whole `scenes[]` as one continuous film via **`@remotion/transitions` `TransitionSeries`**: a `.Sequence` per scene (length = `duration_frames`) plus, before every scene after the first whose `transition_in` is non-`cut`, a `.Transition` of `transition_in.duration` overlap. `fade`/`wipe`/`slide` use their dedicated `@remotion/transitions` presets; `iris`/`mask`/`morph-match`/`match-cut`/`camera-continuous` fall back to `fade()` for now (proper SVG-mask / flubber / shared-element compositors are later-milestone work). Every transition is timed by `linearTiming({ easing: StyleKit "smooth" bezier })` so no transition is linear (spec §9). A `cut`/absent transition emits NO `.Transition` (segments butt = hard cut).
- **Total-duration computation.** `config.duration_frames = Σ scene.duration_frames − Σ transition overlaps` (computed in the lowering pass; `DEFAULT_TRANSITION_FRAMES = 15` when a non-`cut` transition omits `duration`). `Root.tsx`'s `calculateMetadata` independently recomputes the same sum and **throws** on mismatch (no silent truncation) — the lowering math and the rendered `TransitionSeries` length must agree.
**Determinism notes:** `TransitionSeries`/`linearTiming` are pure functions of Remotion's frame clock; the procedural provider is SVG/DOM, so the whole film stays deterministic on `gl:'angle'` (no software-GL disk bloat). Verified per the verify-render skill: rendered `projects/blip-story` (4 scenes — fade·fade·slide, 366f) in TWO separate processes and the **decoded video streams were byte-identical** (`ffmpeg -map 0:v -f md5`); frames extracted to /tmp and visually confirmed (fade crossfade at the b1→b2 boundary, slide seam at b3→b4). `projects/neuron-story` (3 scenes) also renders. typecheck clean.
**Files:** schema already reserved the fields (`src/ir/scene.ts` `TransitionSchema`/`transition_in`/multi-scene `scenes[]`; `src/ir/story.ts` `StoryTransitionSchema`/beat `transition`+`duration`); this milestone wired the lowering, compositor, and Root cross-check.

## 2026-06-22 — `scatter` generator: procedural density (§10.1)
**Context:** The Kurzgesagt look needs "hundreds of tiny shapes" density — starfields, dust, foliage, sparkle, crowds — as an ambient layer that composites over rigs/other generators. Keyframes and the bead-string can't express this; §10.1 called for a `scatter` generator + a shape budget.
**Decision:** Added `src/generators/scatter.tsx` as the second generator, following the bead-string contract exactly: a React `FC<GeneratorComponentProps>` that reads `useCurrentFrame()`/`useVideoConfig()` unconditionally then overrides with props, delegating to a **pure `renderScatter(props)`** (unit-testable, no hooks). Params are loose at the IR boundary and narrowed by `ScatterParamsSchema` (`.strip()`, every field defaulted) parsed at render time; colors resolve via `resolveFill` (palette token OR hex). Registered under key `'scatter'` in `registry.ts` (the only wiring step) and exported from `generators/index.ts`.
- **Placement:** `distribution: 'grid'` (default) is a near-square jittered even grid (cols/rows ≈ √(count·aspect), per-cell seeded jitter = poisson-disc-like even coverage without a mechanical lattice); `'random'` is seeded uniform. Region defaults to the full composition; any of `{x,y,w,h}` may be set to bound a sub-area.
- **Per-element variation:** each element draws size/color/rotation/animation-phase from its own `mulberry32(mixSeed(seed, i+1))` sub-stream; placement uses a separate `mixSeed(seed,'plac')` stream so the two never correlate.
- **Motifs:** `dot` (`<circle>`), `star` (a 4-point sparkle `<path>`, translate+rotate), `blob` (reuses `path.ts` `blobPath` for an organic closed path with its own seeded wobble).
- **Animation:** per-element `twinkle` (opacity), `pulse` (scale), `drift` (Lissajous wander), `none` — all driven purely by `frame + phase` at `anim_speed` Hz (deterministic, neighbours out of step). Optional `glow` (SVG gaussian-blur halo) and layer `opacity`.
- **Shape budget (§10.1):** `SCATTER_SHAPE_BUDGET = 2000` (in `types.ts`); `count` above it is **clamped and `console.warn`-ed with the exact drop count** — never silently truncated (matches the "no silent caps" rule). Verified: count=5000 → 2000 rendered + a warning.
**Determinism:** pure function of (params, seed, frame); no `Date.now`/`Math.random`. Rendered as an absolutely-positioned full-size `<svg>` so it composites cleanly. Verified per `verify-render` on `projects/scatter-demo` (3 scenes: star starfield · dot dust · blob foliage, 264f): two SEPARATE cold renders (caches cleared) gave byte-identical **decoded** video streams (`ffmpeg -map 0:v -f md5` = `012293749759f11ef66ac81c869c3359`); frames extracted to /tmp and visually confirmed all three motifs + animations composite over the bead-string + rig. Render uses `gl:'angle'` (SVG/DOM determinism; software GL stays banned — disk balloon).
**Rationale:** One more registered module, zero IR/pipeline changes — proves the §10 "adding a generator = adding one module" claim a second time and lands the §10.1 procedural-detail half of the object-detail strategy (baking + Pixi instancing remain).

## 2026-06-22 — First-class `shape` layer: @remotion/shapes primitives + flubber morph (ADR-003 #1)
**Context:** ADR-003 backlog #1 (Tier A, deterministic). The `shape` layer was a placeholder (`src/render/Scene.tsx` rendered an empty `AbsoluteFill`); `MorphChannelSchema`/`GradientFillSchema` existed but were unused. "Families are sockets; libraries are plugs" — so the shape socket adopts free libraries rather than hand-rolling vector geometry/morphing.
**Decision (libraries adopted):**
- **`@remotion/shapes`** — the PRIMITIVE catalog. A `shape: { kind, …params }` descriptor (extended `ShapePrimitiveSchema`, `kind` ∈ rect/circle/ellipse/triangle/star/polygon/pie/heart, params `.passthrough()`) selects the matching maker in `geometryForPrimitive`; per-kind defaults keep a minimally-authored layer renderable. The maker returns a pure path `d` + intrinsic box → SVG `viewBox`.
- **`flubber`** — PATH MORPH. The `morph` channel (`animated(string)` of SVG `d` strings) is evaluated in `evalMorph`: locate the active `[a,b]` keyframe segment for the frame, then `flubber.interpolate(a.d, b.d, {maxSegmentLength:10})` at the StyleKit-eased segment progress. flubber resamples DIFFERING point counts (star→heart works). `morph` takes precedence over a `shape` primitive when both are present. flubber ships UMD with no real ESM entry, so we import the namespace and probe `.interpolate ?? .default.interpolate` to work under both the Remotion webpack bundle and tsx/Node.
- **`@remotion/paths`** — `getBoundingBox` frames the morph path (its coords can start anywhere) into the SVG viewBox.
**Fill / stroke:** `fill` is a solid (animated) colour OR a linear/radial gradient. Gradients lower to an SVG `<defs>` `<linearGradient>`/`<radialGradient>` whose id is `shape-grad-${layer.id}` — DETERMINISTIC from the layer id, so two shapes never collide and markup is byte-identical every run. Linear angle (deg) maps to objectBoundingBox unit-vector endpoints. Palette tokens resolve through `resolveFill` (shared with the generator family); animated colours interpolate with Remotion `interpolateColors`. `stroke` is an optional colour+width. The `{a,k}` transform (position/scale/rotation/opacity) is evaluated with the same `eval.ts` StyleKit easing as every other layer.
**Wiring:** `src/render/Scene.tsx` dispatches `'shape'` → `<ShapeLayer>` inside the same `ParallaxWrapper` + §11.1 shading/effects wrappers as rig/generator, so shapes get depth + shading identically. `<ShapeLayer>` stays presentational (parallax/shading owned by `<Scene>`).
**Determinism:** pure SVG/DOM — no Pixi, no software GL; renders byte-identical on `gl:'angle'`. flubber is a deterministic geometric interpolator; gradient ids are content-derived. Verified per `verify-render`: typecheck clean; two SEPARATE cold render processes of `projects/shapes-demo` (2 scenes — a 5-primitive constellation with solid/linear/radial fills + strokes, then a star→heart morph with fill tracking gold→blue, 162f) gave byte-identical **decoded** video streams (`ffmpeg -map 0:v -f md5` = `3e83a6852fcb8d17dd8f6c14f7bfa5bd`); frames extracted to /tmp (primitives @30, mid-morph @92 = a genuine flubber in-between, heart @120) and visually confirmed.
**Files:** `src/ir/scene.ts` (already had `ShapeLayerSchema` with `shape`/`morph`/`fill`/`stroke`/`transform` reserved) · `src/render/ShapeLayer.tsx` (new renderer) · `src/render/Scene.tsx` (dispatch) · `examples/shapes-demo.yaml` + `projects/shapes-demo` (vertical slice) · deps `@remotion/shapes`/`@remotion/paths`/`flubber` in `package.json`.
**Rationale:** Adopting catalogs into the family (zero engine/IR change beyond the reserved descriptor) lands shapes + morph as a shippable, deterministic Tier-A increment, consistent with ADR-001/ADR-003.
